Use case diagram



What a "use-case" is?

In essence, a use-case outlines a sequence of actions or steps performed by a user or actor (which could be a person, another system, or an external entity) to achieve a specific goal within the system. Use-cases typically describe the interactions between the user and the system and can range from simple tasks to complex workflows.

Enumerate the elements that may be represented in a UML use-case diagram.

In a UML (Unified Modeling Language) use-case diagram, various elements can be represented the interactions between actors and use-cases within a system. Here are the key elements typically found in a UML use-case diagram:

Click to view



Actor

An actor represents an external entity (such as a user, system, or another application) that interacts with the system.

Click to view



Use Case

It describes a sequence of interactions between the actor(s) and the system to achieve a particular goal.

Click to view



System Boundary

The system boundary defines the scope of the system being modeled. It separates the system from its external actors and environments.

Click to view



Relationships


Association

An association represents a relationship between an actor and a use case, indicating that the actor interacts with the system to perform the actions described in the use case.


Generalization

Generalization is used to represent inheritance relationships between use cases. It indicates that one use case inherits behavior from another.


Include Relationship

An include relationship denotes that one use case includes the behavior of another use case. It signifies that the included use case is a part of the behavior of the including use case.


Extend Relationship

An extend relationship signifies that one use case extends the behavior of another use case under certain conditions. It indicates optional behavior that may be added to the extended use case.

Create UML use-case diagram(s) to describe the interaction between the user and your BMI system, which you have/will create for your Android native application.

                        
@startuml
left to right direction
skinparam packageStyle rectangle
"BMI APP" as mysystem
actor USER
rectangle mysystem {
USER --> (BUTTON) : click
USER --> (settings) : click
USER --> (INPUT) : Height & Weight
(test USER data) .> (calculate BMI)
(calculate BMI) .> USER: calculated BMI
(INPUT) . (test USER data)
}

settings .> SETTINGS
rectangle SETTINGS {
USER --> (SWITCH) : click
SWITCH .> (ColorTheme) : click
(ColorTheme) .> USER: ColorTheme
USER --> (OnOff) : click
OnOff .> (CountSteps) : click
CountSteps .> (TotalAmount) : show
TotalAmount .> USER: TotalAmount
}
@enduml
